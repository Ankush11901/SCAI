## 1. Article Generation Workflow

### Generation Steps

```
1. SELECT TYPE       →  Choose 1 of 9 article types
        ↓
2. CONFIGURE         →  Set optional toggles on/off
        ↓
3. DATA INPUT        →  Enter keyword/topic
                        Pre-fill external data (if applicable)
        ↓
4. HEADER TYPE       →  Select: Question / Statement / Listicle
        ↓
5. GENERATE          →  LLM generates content
        ↓
6. VALIDATE          →  Check all rules compliance
```

### Article Type Selection Logic

- One article = one type
- Cannot mix article types within a single article
- Type determines required components
- Type determines content flow sequence
- Type determines available interlinking strategies

### Nine Article Types

1. Affiliate
2. Commercial
3. Comparison
4. How-To
5. Informational
6. Listicle
7. Local
8. Recipe
9. Review

## 2. Content Source Rules

### LLM-Generated Content

| Category | What LLM Generates |
|----------|-------------------|
| **Universal Components** | H1, H2s, Overview Paragraph, Standard Paragraphs, Closing H2, Closing Paragraph |
| **Meta Content** | Meta Title, Meta Description |
| **Alt Text** | Featured Image Alt Text, H2 Image Alt Text |
| **All Paragraphs** | Prose content for all paragraph types |
| **All Lists** | Bullets, numbered items (except external data) |

### Meta Content Generation Sources

| Component | Generated From |
|-----------|----------------|
| **Meta Title** | H1 + Primary keyword optimization |
| **Meta Description** | H1 + Overview Paragraph summary |
| **Featured Image Alt Text** | H1 and image context |
| **H2 Image Alt Text** | H2 heading it's tied to |

### Article-Specific LLM Content

| Article Type | LLM Generates |
|--------------|---------------|
| Review | Features List, Pros & Cons Lists, Rating score + justification |
| Recipe | Nutrition Table content (with "approximate values" disclaimer) |
| Commercial | Feature List items, CTA Box text |
| Informational | Key Takeaways Box items, Quick Facts List items |
| How-To | Materials/Requirements Box items, Pro Tips List items |
| Comparison | Topic Overviews, Comparison Table content, Quick Verdict |
| All Types | All bulleted lists matching word count requirements |

### External/Pre-filled Content

Content NOT generated by LLM:

| Article Type | Content Source | Data |
|--------------|----------------|------|
| **Affiliate** | Amazon/External API | Product Card: Image, Name, Rating, Price, CTA Button |
| **Local** | User Input Fields | Service Info Box: Hours, Area, Phone, License, Years in Business |

**Important**: Only Affiliate uses external API. Review is 100% LLM-generated (no API).

### Auto-Generated Content

Content automatically generated by the system:

| Content | Generation Method |
|---------|-------------------|
| Table of Contents | Auto-generated from all H2 headings |
| Header Types | Determined by H1 type selection |
| Listicle Numbers | Auto-numbered based on count and order setting |

### Content Adaptation from External Data

- **Affiliate H2**: Generated from product name, adapted to match H1 type
- **Affiliate Paragraph**: Describes product features based on external data
- **All Content**: Must be relevant to provided keyword/topic

## 3. Header Consistency and Adaptation

### CRITICAL RULE: H1 Type Determines ALL H2 Types

```
┌─────────────────────────────────────────────────────────────┐
│  If H1 is X type, ALL H2s must be X type too!              │
│                                                             │
│  NO EXCEPTIONS - Applies to ALL H2s including unique        │
│  component H2s (FAQ, Rating, Features, etc.)               │
└─────────────────────────────────────────────────────────────┘
```

### Header Type Mapping

| H1 Type | ALL H2s Must Be | Example H2 |
|---------|-----------------|------------|
| Question | Question format | "What Is This?", "How Does It Work?" |
| Statement | Statement format | "Understanding the Basics", "Key Features" |
| Listicle | Listicle format | "1. First Point", "2. Second Point" |

## 4. Content Sequencing

### Universal Content Flow

All articles follow this base flow:

```
H1
  ↓
Featured Image
  ↓
Overview Paragraph (elaborates on H1)
  ↓
[Key Takeaways Box - Informational only, after Overview]
  ↓
[Table of Contents - optional, after Overview]
  ↓
MAIN CONTENT LOOP
  ├── H2
  ├── H2 Image (optional)
  └── Standard Paragraph (elaborates on H2)
  ↓
[SPECIAL SECTIONS - article-type specific]
  ↓
Closing H2 + Closing Paragraph (when applicable)
  ↓
FAQ Section (optional)
```

### Article-Specific Sequences

#### Affiliate Sequence
```
H1 → Featured Image → Overview → [Product Card → H2 → H2 Image (opt) → Standard Paragraph] (loop for each product) → Closing H2 + Closing Paragraph (optional)
```

#### Commercial Sequence
```
H1 → Featured Image → Overview → Feature H2 → Feature List → CTA Box → [H2 → H2 Image (opt) → Standard Paragraph] (loop)
```

#### Comparison Sequence
```
H1 → Featured Image → Overview → [Topic H2 → Topic Image (opt) → Topic Overview] (repeat for 2+ topics) → Comparison Table → [H2 → H2 Image (opt) → Standard Paragraph] (loop) → Quick Verdict Box (opt) → Closing H2 → Closing Paragraph
```

#### How-To Sequence
```
H1 → Featured Image → Overview → Materials/Requirements H2 → Materials/Requirements Box → [Step H2 → H2 Image (opt) → Standard Paragraph] (5-10 steps) → Pro Tips H2 + Pro Tips List (opt) → Closing H2 → Closing Paragraph
```

#### Informational Sequence
```
H1 → Featured Image → Overview → Key Takeaways Box → [H2 → H2 Image (opt) → Standard Paragraph] (loop, min 4) → Quick Facts H2 (opt) → Quick Facts List (opt) → Closing H2 → Closing Paragraph
```

#### Listicle Sequence
```
H1 → Featured Image → Overview → [H2 Item → H2 Image (opt) → Standard Paragraph] (odd number loop) → Honorable Mentions H2 (opt) → [H3 → Paragraph] (3-4x)
```

#### Local Sequence
```
H1 → Featured Image → Overview → [H2 → H2 Image (opt) → Standard Paragraph] (loop) → Why Choose Local H2 (opt) → [Image + List] → Service Info Box (opt)
```

#### Recipe Sequence
```
H1 → Featured Image → Overview → Ingredients H2 → Ingredients List → Instructions H2 → Instructions List → Tips H2 → Tips Paragraph → Nutrition Facts H2 (opt) → Nutrition Table (opt) → [H2 → H2 Image (opt) → Standard Paragraph] (loop) → [FAQ H2 → 5 FAQ H3s + Answers] (opt)
```

**Recipe FAQ Example Questions:**
When FAQ is enabled for Recipe articles, common questions include:
- "Can I freeze this?"
- "What substitutes work?"
- "How long does it last?"
- "How to reheat?"
- "Best storage method?"
- "Can I make this ahead?"
- "What sides go well with this?"
- "How to adjust for dietary restrictions?"

**Instructions List Note:**
Word count is flexible (150-400 words) - only exceeds 150 when recipe complexity requires additional steps.

#### Review Sequence
```
H1 → Featured Image → Overview → Features H2 → Features List → Pros & Cons H2 → Pros & Cons Lists → [H2 → H2 Image (opt) → Standard Paragraph] (loop) → Rating H2 → Rating Paragraph → Closing H2 → Closing Paragraph
```

**Review H2 Loop Suggested Topics:**
The H2 + Standard Paragraph loop in Review articles can cover:
- Performance analysis
- Design and build quality
- User experience
- Comparisons to competitors
- Value analysis (price vs features)
- Use cases and applications
- Target audience suitability
- Long-term durability
- Setup and configuration
- Support and warranty

*Minimum 3-5 H2 sections recommended in loop for comprehensive coverage.*

**Review Use Cases:**
Review article type is suitable for all review types:
- Products
- Services
- Experiences
- Software
- Editorial reviews

## 8. Word Count Distribution Logic

### Distribution Principle

Target word count distributes in this priority:

1. **Required components first** - Fixed word counts from component definitions
2. **H2 + Standard Paragraph loops** - Variable, fills remaining word count
3. **Optional components** - Added when enabled by user

### H2 Loop Scaling

- H2 + Standard Paragraph loop minimum: 1 iteration
- Loop expands based on target word count
- Each loop iteration adds approximately the Standard Paragraph word count
- System calculates iterations needed to meet target

### Word Count Calculation Example

```
Target: 2000 words

Fixed components:
- Overview Paragraph: 100 words
- Closing Paragraph: 50 words
- Key Takeaways Box: 75 words (if Informational)
= ~225 words fixed

Remaining for loops:
2000 - 225 = 1775 words

Standard Paragraph = 150 words
1775 ÷ 150 ≈ 12 H2 + Paragraph iterations
```

### Content Tier Logic

| Tier | Typical Word Count | Use Case |
|------|-------------------|----------|
| Primary | ~2000 words | Pillar content, competitive keywords, comprehensive guides |
| Secondary | ~1000 words | Supporting content, long-tail keywords, topic clusters |

## 9. Quality Standards

### Paragraph Writing Principles

- Written clearly as complete paragraphs
- Must conclude with a period
- Address all key points of the heading it belongs to
- Writing must be clear and well-organized
- Proofread to eliminate errors in grammar, spelling, or punctuation
- Goal: Provide clear and informative response that conveys understanding

### Natural Language Principles

- Content should sound natural and conversational, not robotic or AI-generated
- Avoid overly formal or mechanical phrasing
- Write as a knowledgeable human would speak to the reader
- Vary sentence structure and length for natural rhythm

### Clarity and Readability

- Use clear, concise language
- Break complex topics into digestible sections
- Maintain consistent tone throughout article
- Ensure logical flow between sections

### Value and Relevance

- Every paragraph must add value
- Content must directly relate to H2 topic
- Avoid filler or redundant information
- Focus on user benefit and practical application

### Actionable Content

- Include actionable tips, steps, or recommendations where appropriate
- Provide concrete examples readers can apply
- Avoid vague or generic advice
- Give readers clear next steps or takeaways

### Accuracy and Authority

- LLM-generated content must be factually accurate
- Review articles require balanced, fair assessment
- How-To steps must be executable and safe
- Recipe nutrition must include "approximate" disclaimer

### Engagement and Scannability

- Use bullet points for easy scanning
- Keep sentences and paragraphs focused
- Highlight key information in boxes/lists
- Structure content for quick comprehension

### Article-Specific Quality Requirements

| Article Type | Quality Requirements |
|--------------|---------------------|
| Affiliate | H2 must contain product name; paragraph describes features |
| Commercial | Feature List scannable; CTA action-oriented |
| Comparison | Topic Overview follows 2-paragraph structure; balanced coverage |
| How-To | Steps clear and sequential; Materials listed upfront |
| Informational | Key Takeaways at top; minimum 4 H2 sections |
| Listicle | Odd numbers only; equal treatment per item |
| Local | Keywords include Service + Location |
| Recipe | Ingredients bulleted; Instructions numbered; Tips practical |
| Review | 100% LLM-generated; balanced Pros & Cons; Rating justified |

## 10. Internal Linking Guidelines

### Overview

Internal linking connects related content within the site, improving both SEO performance and user navigation.

### 10.1 Purpose of Internal Linking

| Purpose | Benefit |
|---------|---------|
| **SEO Value** | Passes link equity, signals content relationships to search engines |
| **User Experience** | Helps readers discover related content |
| **Topical Authority** | Builds semantic clusters around key topics |
| **Engagement** | Keeps users on site longer |

### 10.2 When to Add Internal Links

| Situation | Link Action |
|-----------|-------------|
| Mentioning a topic covered elsewhere | Link to that article |
| Referencing a service/product | Link to service page (Commercial/Local only) |
| Discussing a location | Link to location page (Local only) |
| No relevant links exist | Use "Related Reading" section as fallback |

### 10.3 Link Integration Rules

| Rule | Description |
|------|-------------|
| **Contextual Relevance** | Links must relate to surrounding content |
| **Natural Flow** | Links should read naturally within sentences |
| **Descriptive Anchor Text** | Use meaningful text, not "click here" or "read more" |
| **Value-Adding** | Only link if it genuinely helps the reader |

### 10.4 What NOT to Do

| Prohibited | Reason |
|------------|--------|
| Forcing irrelevant links | Harms user experience and SEO |
| Over-linking | Dilutes link value, looks spammy |
| Linking same page multiple times | Redundant, no added value |
| Generic anchor text | "Click here" provides no SEO value |

## 11. Content Safety Rules

### Overview

Content safety rules ensure all generated content is appropriate, professional, and free from harmful material.

### 11.1 No Adult Content (ENFORCED)

**Rule**: All generated content must be appropriate for general audiences.

**Prohibited**:
| Category | Description |
|----------|-------------|
| Sexual content | Sexually explicit or suggestive text |
| Graphic violence | Detailed descriptions of violence, gore, or injury |
| Drug content | Promotion or detailed instructions for illegal drug use |
| Gambling | Promotion of gambling without appropriate disclaimers |
| Weapons | Instructions for creating weapons or explosives |

**Enforcement**: Content flagged for adult themes triggers regeneration with stricter content filters.

---

### 11.2 No Profanity (ENFORCED)

**Rule**: All generated content must be free from profanity and offensive language.

**Prohibited**:
| Category | Description |
|----------|-------------|
| Swear words | Any profane language in any language |
| Slurs | Racial, ethnic, gender, or any discriminatory slurs |
| Crude language | Vulgar or crude expressions |
| Insults | Direct insults or demeaning language toward any group |
| Hate speech | Any language promoting hatred or discrimination |

**Allowed Alternatives**:
- Use professional, neutral language
- Replace with appropriate synonyms
- Omit rather than substitute with symbols (e.g., no "f***")

---

### 11.3 No Negative Language (ENFORCED)

**Rule**: Avoid unnecessarily harsh, harmful, or discouraging phrasing.

**Prohibited Patterns**:
| Pattern | Example | Better Alternative |
|---------|---------|-------------------|
| Absolute negatives | "You will never succeed" | "Success requires practice" |
| Discouraging language | "This is too hard for beginners" | "Beginners may need extra time" |
| Fear-mongering | "You'll fail without this" | "This can help improve results" |
| Harsh criticism | "This product is terrible" | "This product has limitations" |
| Personal attacks | "Only fools would..." | "Consider these factors..." |

**Appropriate Use of Negative Language**:
- Cons sections in reviews (factual, balanced)
- Warning sections (safety-focused)
- Comparison disadvantages (objective)

**Tone Guidelines**:
| Instead of | Use |
|------------|-----|
| "Don't" | "Consider avoiding" or "It's best to" |
| "Wrong" | "Less effective" or "Not recommended" |
| "Stupid" | "Not advisable" or "Reconsidered" |
| "Never" | "Rarely" or "In most cases, avoid" |

---

### 11.4 Narrative Consistency (ENFORCED)

**Rule**: Content must not contain contradicting statements within the same article.

**Definition**: Narrative consistency means the article maintains a coherent message without conflicting claims.

**Prohibited Contradictions**:
| Type | Example | Why It's Wrong |
|------|---------|----------------|
| Sentiment flip | "This is excellent" → "This is disappointing" | Confuses reader |
| Fact contradiction | "Costs $50" → "Very affordable at $500" | Misleading |
| Recommendation conflict | "Highly recommend" → "Not worth buying" | Inconsistent advice |
| Feature contradiction | "Has 10 features" → "Limited with only 3 features" | Factually inconsistent |

**Allowed Nuance**:
| Situation | How to Handle |
|-----------|---------------|
| Pros and Cons | Clearly label sections, maintain balance |
| Different use cases | "For beginners: X, For experts: Y" |
| Conditional statements | "If budget is priority, then... If quality is priority, then..." |
| Updates/Changes | "Previously X, now updated to Y" |

**Consistency Checks**:
- Compare sentiment across all paragraphs
- Verify numerical facts don't contradict
- Ensure recommendations align throughout
- Check that comparisons are balanced

**When Inconsistency Detected**:
1. Identify the contradicting statements
2. Determine which is accurate/intended
3. Revise or regenerate the inconsistent content
4. Re-validate entire article for consistency

---

### 11.5 Content Safety Summary

| Rule | Enforcement Level | Action on Violation |
|------|-------------------|---------------------|
| No Adult Content | Strict | Regenerate entire section |
| No Profanity | Strict | Regenerate with filter |
| No Negative Language | Moderate | Revise phrasing |
| Narrative Consistency | Moderate | Identify and correct contradictions |
