# Engineering Standup Breakdown (Taofeek, Mustapha, Kalu)

## 1) Product/Billing Decisions (Finalized for Build)

### Base Plan
- **Plan Price:** `$99/month`
- **Included Usage Target:** `100,000 words`
- Word usage maps to **LLM token cost** (provider charges per token).

### Image Usage Assumption (for quota modeling)
- Working assumption: **3–4 images per 1,000 words**
  - ~1 featured image
  - ~2–3 in-content images
- For `100,000 words`, projected image range is roughly:
  - `300–400 images` (strict 3–4 math)
  - Team discussed a broader guardrail up to ~`500` as buffer.

### Quota Logic
- The `$99` cap should represent a **combined budget envelope**:
  - text generation cost + image generation cost
- System should enforce usage so monthly included value stays within plan economics.

### Pay-As-You-Go (Overage)
- Once user exceeds included quota, overage must follow the **same base exchange ratio** derived from:
  - `$99 : 100,000 words : token cost model`
- Image model switching (e.g., Flux vs Gemini image options) must preserve **equivalent margin rate**.
- Implementation principle:
  - normalize everything to a baseline unit cost
  - apply model-specific multipliers
  - keep revenue consistency regardless of model chosen

## 2) WordPress Plugin Clarifications

- WordPress plugin installability is primarily **permissions-based**.
- Users can install plugins if:
  - plugin is available/allowed in their environment
  - their role has sufficient permissions (admin-level typically)
- Managed WP environments may restrict:
  - installs outside approved/plugin directory channels
- If user lacks permission, show explicit error:
  - _“You don’t have permission to install plugins. Please contact your site administrator.”_
- Requirement for this sprint:
  - ensure plugin is visible
  - can be installed (where authorized)
  - can be tested end-to-end
